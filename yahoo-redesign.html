UX Design · Landing Page · Conversion Optimisation

Redesigning the
Yahoo Landing Page

A UX-driven redesign of Yahoo's cluttered homepage — rethinking information architecture, visual hierarchy, and layout structure to reduce cognitive load, improve content discoverability, and create a page experience that actually motivates users to scroll and engage.

Project type

Concept Redesign

Focus

UX · Layout · CRO

Tools

Figma · Adobe XD

Scope

UX Design Landing Page CRO

01 — The Problem

When everything competes,
nothing converts

Yahoo's existing landing page presents users with an overwhelming volume of content the moment they arrive — news, advertisements, weather widgets, email previews, trending topics, and promotional banners all fighting for attention simultaneously. The result is a page that feels chaotic rather than useful.

The core UX problem is the absence of visual hierarchy. Without a clear reading order, users scan aimlessly rather than following an intentional path. This directly impacts dwell time, scroll depth, and content engagement — the three metrics that determine whether a homepage is working.

Problems identified

  • Excessive information density on first load creates cognitive overload
  • Multiple competing visual focal points — no clear hierarchy
  • Advertisements and editorial content indistinguishable at a glance
  • No user control over what content appears in their view
  • Layout discourages the first scroll — users leave without exploring
  • Visual noise undermines trust in the content being presented

Design goals

  • Reduce visual noise while preserving the same volume of content
  • Establish a clear layout grid with intentional content zones
  • Separate advertising from editorial through spatial structure
  • Introduce user personalisation to drive return engagement
  • Create a first fold that motivates scroll without overwhelming
  • Use colour purposefully — CTAs stand out, content breathes
Before — Existing Yahoo Homepage

The original Yahoo landing page — competing elements, no clear visual hierarchy, high cognitive load on arrival.

02 — Design Decisions

Four principles that drove
every layout choice

The redesign wasn't about stripping Yahoo of its content richness — it was about organising that richness so users could navigate it on their own terms. Every decision below was made with one question in mind: does this make it easier or harder for a user to find what they're looking for?

Decision 01

Structured grid — content in its lane

All content is placed on a consistent layout grid. Advertisements are given a defined space rather than being scattered throughout, so users always know what they're looking at. News, weather, and finance sections each occupy distinct zones — the layout does the organisational work so users don't have to.

Decision 02

News carousel — same content, less clutter

Rather than stacking all news items vertically, a horizontal carousel keeps the content volume intact while dramatically reducing the page's vertical density. Users access the same amount of news on their own terms, scrolling through the carousel only when they choose to engage.

Decision 03

Customise Home — user-led personalisation

A "Customise Home" feature was introduced, allowing users to choose which widgets appear on their landing page. This shifts the experience from a one-size-fits-all broadcast to a personalised dashboard — increasing relevance, reducing noise, and giving users a reason to return.

Decision 04

Blue CTAs — colour as a conversion signal

Additional information and action links are highlighted in Yahoo's brand blue — used exclusively for calls to action. This creates a consistent visual language where colour carries meaning: blue means "there's more here." Users learn this pattern quickly, which improves content discovery and click-through behaviour.

UX principle applied: Maintaining the same information volume while reducing perceived complexity. The goal was never to simplify Yahoo — it was to make its depth feel navigable rather than overwhelming.

03 — The Redesign

A cleaner page that carries
the same content

The redesigned landing page retains Yahoo's full content offering — news, ads, weather, finance, trending topics — but organises it within a structured grid that creates natural reading flow. The first fold is intentionally cleaner, giving users a clear entry point before the page opens up into categorised sections below.

Structured ad zones — Advertising has a fixed position in the layout, clearly separated from editorial content spatially rather than visually.

News carousel — Horizontal scrolling for news reduces page height while keeping full content breadth accessible to engaged users.

Customise Home widget — Persistent in the header, giving users agency over their experience without requiring a settings detour.

Blue CTA system — A consistent colour signal applied to all secondary actions, improving content discovery and maintaining brand identity.

After — Redesigned Landing Page

The redesigned page — consistent grid, defined content zones, carousel for news density, blue CTA system, and Customise Home for user personalisation.

The takeaway

"A landing page doesn't fail because it has too much content — it fails because that content has no order. This redesign was an exercise in giving structure to complexity: proving that the same information, laid out with intention, can feel like a completely different experience."
